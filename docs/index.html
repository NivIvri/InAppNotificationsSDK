InApp Notifications SDK

Course project — Android in-app notification system with a backend and admin portal.

Overview Quickstart API Reference

What Is This?

This project is an Android SDK that shows notification dialogs inside an app. The notifications are stored in Firestore and managed through a web admin portal. You can create, edit, and activate notifications without changing the app code.

How It Works

  1. You add the SDK to an Android app and call checkAndShow() on any screen
  2. The SDK calls the backend with the screen name and user type
  3. The backend checks Firestore for active, valid notifications that match
  4. If one is found, it's shown as a dialog
  5. The SDK tracks views and clicks back to the backend

Components

notification-sdk

The Android library module. Contains the SDK code that gets included in any Android app.

app

A demo Android app that shows how to use the SDK on different screens with different user types.

backend

A Flask REST API that reads and writes notifications to Firestore. Also handles view and click tracking.

admin-portal

A plain HTML/JS web page to create, edit, activate/deactivate, and delete notifications. Connects directly to the backend.

Technologies Used

  • Kotlin — Android SDK and demo app
  • Retrofit 2 — HTTP client for the SDK
  • Python + Flask — Backend API
  • Google Firestore — Database
  • Firebase Admin SDK — Backend Firestore access
  • HTML / CSS / JavaScript — Admin portal

Known Limitations

  • No API authentication — the backend endpoints are open
  • showOnce is per-device, not per-user account
  • Only one notification is shown per screen visit
  • Backend is not deployed — runs locally only
  • No tests written
The serviceAccountKey.json file is excluded from Git. You need to generate your own from Firebase Console to run the backend.
