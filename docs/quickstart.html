Quickstart

Follow these steps to get everything running locally.

Overview Quickstart API Reference

1. Set Up Firebase

1

Go to console.firebase.google.com and open your project.

2

In the left sidebar: Build → Firestore Database → Create database

Choose Start in test mode, pick a region, click Enable.

3

Go to Project Settings → Service Accounts → Generate new private key

Save the downloaded file as backend/serviceAccountKey.json

Never commit serviceAccountKey.json — it's already in backend/.gitignore.

2. Run the Backend

1

Open a terminal and go to the backend/ folder.

2

Create and activate a virtual environment:

python -m venv venv
venv\Scripts\activate.bat
3

Install dependencies:

pip install -r requirements.txt
4

Start the server:

python app.py

You should see: Running on http://127.0.0.1:5000

3. Run the Admin Portal

Open admin-portal/index.html directly in a browser. No server needed.

The portal connects to http://localhost:5000 automatically.

1

Fill in the form to create a notification. Example values:

Title:         Welcome!
Message:       Thanks for using our app.
Target Screen: home
Audience:      all
Start Date:    (leave empty — defaults to today)
End Date:      (leave empty — no expiry)
2

Click Create. The notification appears in the table below.

4. Run the Android App

1

Open the project in Android Studio.

2

In app/src/main/.../demo/MainActivity.kt, check the BASE_URL:

// For emulator:
const val BASE_URL = "http://10.0.2.2:5000/"

// For real device (use your PC's local IP):
const val BASE_URL = "http://192.168.1.x:5000/"
3

Run the app. Select a user type and tap Go to Home Screen.

The notification dialog should appear.

If the notification appears — everything is working correctly.

SDK Usage in Your Own App

To use the SDK in a different Android project:

Step 1 — Initialize (once, in your main activity)

NotificationSdk.initialize("https://your-backend-url/")

Step 2 — Show notifications on a screen

// Call this in onResume() of any activity
NotificationSdk.checkAndShow(this, "home", "regular")

screen — matches the targetScreen field you set in the admin portal
userType — "regular", "premium", or "all"

Step 3 — Reset seen notifications (optional)

NotificationSdk.clearSeenNotifications(this)
